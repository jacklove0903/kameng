# 视频聚合生成工具 — 功能规划

## 一、核心功能

### 1. 多窗口并行生成
- 最多 **4 个窗口**同时工作，每个窗口独立配置
- 每个窗口可选择**不同的模型提供商**（如 Runway、Pika、Kling、Sora、Stable Video Diffusion 等）
- 窗口布局可切换：2×2 / 1+3 / 4 横排 / 全屏单窗口
- 各窗口生成进度**独立显示**（进度条、预计剩余时间）
- 支持**暂停/取消**单个窗口的生成任务

### 2. 模型提供商管理
- **提供商注册**：添加/编辑/删除模型提供商
- 每个提供商配置项：API Key、API Endpoint、模型名称、支持的参数范围
- **提供商状态检测**：实时检测 API 可用性、剩余额度
- 内置常见提供商模板（Runway Gen-3、Kling、Pika、Luma 等），减少手动配置
- 统一抽象层：不同提供商的参数差异由系统做映射转换

### 3. 视频生成参数配置
每个窗口独立配置：

| 参数 | 说明 |
|------|------|
| **Prompt** | 正向提示词 |
| **Negative Prompt** | 反向提示词 |
| **参考图/首帧** | Image-to-Video 模式上传参考图 |
| **尾帧** | 指定视频结束画面 |
| **视频时长** | 根据模型能力选择（如 4s / 5s / 10s） |
| **分辨率** | 720p / 1080p / 4K（取决于模型支持） |
| **宽高比** | 16:9 / 9:16 / 1:1 / 自定义 |
| **运动强度** | 控制画面运动幅度 |
| **种子值** | 可固定种子以复现结果 |
| **采样步数/质量** | 部分模型支持 |

### 4. Prompt 工作流
- **统一 Prompt 输入**：写一次 Prompt，一键分发到所有窗口（可选）
- **Prompt 模板库**：保存常用 Prompt 片段，支持变量替换（如 `{subject}`, `{style}`）
- **Prompt 优化建议**：根据所选模型提供商，提示该模型偏好的 Prompt 风格
- **历史 Prompt 记录**：自动保存，可快速复用

---

## 二、结果管理

### 5. 视频对比与筛选
- **并排对比**：生成完成后 4 个结果同屏播放，逐帧对比
- **同步播放**：多个视频同步播放/暂停/进度拖动，方便对比
- **评分/标记**：对生成结果打星（1-5）或标记为"收藏"/"废弃"
- **帧级预览**：逐帧查看，截图单帧

### 6. 视频后处理
- **下载**：单个下载 / 批量下载（打包 zip）
- **格式转换**：MP4 / GIF / WebM
- **基础裁剪**：裁剪时长、画面区域
- **拼接**：将多个生成结果拼接为完整视频
- **超分/插帧**：调用外部工具提升画质或帧率

### 7. 项目与历史管理
- **项目概念**：一组相关的生成任务归为一个项目
- **生成历史**：按时间线查看所有生成记录，含完整参数
- **收藏夹**：跨项目收藏最佳结果
- **导出报告**：导出某项目的生成参数 + 结果摘要（方便团队分享）

## 三、工作流增强

### 8. 批量与自动化
- **批量模式**：上传一组 Prompt 列表（CSV/TXT），自动排队逐批生成
- **A/B 测试模式**：同一个 Prompt 自动分发到 4 个不同模型，横向对比效果
- **变体生成**：基于一个好结果，自动微调参数（种子、运动强度等）生成变体
- **定时生成**：设置在低峰时段（如凌晨）自动执行队列任务

### 9. 图生视频工作流
- **故事板模式**：上传多张关键帧图片，指定每段的时长，自动生成过渡视频
- **参考图风格迁移**：上传风格参考图，引导生成风格一致的视频
- **多图输入**：首帧 + 尾帧 + 风格参考图，多维度控制

### 10. Chain（链式生成）
- **视频续写**：将一段生成结果的尾帧作为下一段的首帧，实现长视频
- **跨模型接力**：A 模型生成粗略版本 → 提取关键帧 → B 模型精细化

## 四、系统功能

### 11. 费用与配额管理
- **实时费用统计**：每次生成消耗的金额，按提供商分类汇总
- **预算控制**：设置每日/每月预算上限，接近时预警
- **配额显示**：各 API Key 的剩余额度、速率限制
- **费用对比**：同样 Prompt 在不同模型的成本对比

### 12. 队列与任务管理
- **任务队列**：可视化展示排队中的任务
- **优先级调整**：拖拽调整生成顺序
- **失败重试**：自动重试（可配置次数），失败原因记录
- **并发控制**：根据各 API 的速率限制自动调度

### 13. 用户系统（如做多用户）
- 登录/注册（支持 OAuth）
- 每个用户独立的 API Key 配置和生成历史
- 团队协作：共享项目、共享 Prompt 库
- 权限管理：管理员/普通用户

### 14. 设置与偏好
- 默认参数预设（每次新建窗口自动填充）
- 主题切换（暗色/亮色）
- 通知设置：生成完成推送（桌面通知 / 声音 / Webhook）
- 语言设置（中/英）

## 五、技术选型（Spring Boot）

| 方面 | 方案 |
|------|------|
| **框架** | Spring Boot 3.x |
| **异步任务** | `@Async` + 自定义线程池，或集成 Quartz 做调度 |
| **任务队列** | Redis + Redisson / RabbitMQ（复杂场景） |
| **WebSocket** | `spring-boot-starter-websocket` 实时推送进度 |
| **数据库** | PostgreSQL（元数据、用户、项目、历史） |
| **缓存** | Redis（任务状态、限流、会话） |
| **文件存储** | MinIO（自建）或对接阿里云 OSS / AWS S3 |
| **提供商适配层** | 策略模式 + `@Qualifier` 注入，每家一个 Adapter |
| **API 文档** | SpringDoc / Swagger |
| **安全** | Spring Security + JWT |
| **前端** | React/Vue + 视频播放器（video.js / hls.js） |

### Spring Boot 架构优势
- **多提供商适配**：策略模式 + `@Qualifier` 轻松切换不同模型实现
- **并发控制**：`CompletableFuture` + 线程池实现 4 窗口并行
- **流式响应**：SSE（Server-Sent Events）或 WebSocket 推送生成进度
- **错误处理**：统一异常处理器映射各 API 差异化错误码
- **生态成熟**：文件上传、Redis 缓存、定时任务、安全认证均有成熟方案

## 六、差异化亮点（可选但有竞争力）

- **智能推荐**：根据 Prompt 内容推荐最适合的模型（"这个 Prompt 风格适合 Runway"）
- **Prompt 打分**：生成前预估 Prompt 质量，提示优化方向
- **结果 AI 评分**：生成后自动用视觉模型评估视频质量，排序推荐
- **协作白板**：团队成员实时看到彼此的生成进度，讨论选哪个
- **API 开放**：提供自己的 API，让别人通过你的平台调用多模型

---

## 七、MVP 建议路径

### Phase 1 — 核心 MVP
1. Spring Boot 项目搭建，提供商适配层抽象（先接入 1-2 家）
2. 单窗口生成 + 基础参数配置（Prompt、时长、分辨率）
3. 视频结果展示与下载

### Phase 2 — 多窗口
4. 4 窗口并行生成 + 独立配置
5. WebSocket 实时进度推送
6. 统一 Prompt 分发

### Phase 3 — 对比与管理
7. 并排对比 + 同步播放
8. 项目/历史管理
9. 费用统计

### Phase 4 — 高级功能
10. 批量模式 + A/B 测试
11. 链式生成 + 故事板
12. 多用户系统
